# خطة اختبار الويب

هذا المستند مخصص لاختبار سلوك Lumina داخل المتصفحات بعد التحسينات الأخيرة في:
- `browse mode`
- `focus mode`
- القراءة بالأسهم
- التنقل السريع بالحروف
- النماذج
- الجداول
- `live regions`

## قبل البدء

1. شغّل Lumina.
2. افتح `Chrome` أو `Edge` أو `Firefox`.
3. ابدأ بصفحة بسيطة ثم انتقل إلى صفحات أعقد.
4. يفضّل فتح `Inspector` أثناء الاختبار لمقارنة ما يُنطق مع الأحداث المسجلة.

## أوامر يجب اختبارها

- `Insert+Space`
- `Escape`
- `Up / Down / Left / Right`
- `Ctrl+Left / Ctrl+Right`
- `H / Shift+H`
- `K / Shift+K`
- `E / Shift+E`
- `B / Shift+B`
- `X / Shift+X`
- `D / Shift+D`
- `T / Shift+T`
- `L / Shift+L`
- `A / Shift+A`
- `F / Shift+F`
- `Insert+T`
- `Insert+W`
- `Insert+S`
- `Insert+R`
- `Insert+B`
- `Insert+Y`
- `Ctrl+Left / Right / Up / Down` داخل الجداول

## مرحلة 1: التصفح الأساسي

جرّب صفحة تحتوي على:
- عناوين
- فقرات
- روابط
- أزرار

اختبر:
- هل `Down Arrow` يقرأ السطر التالي؟
- هل `Up Arrow` يقرأ السطر السابق؟
- هل `Right Arrow` يقرأ الحرف التالي؟
- هل `Left Arrow` يقرأ الحرف السابق؟
- هل `Ctrl+Right` يقرأ الكلمة التالية؟
- هل `Ctrl+Left` يقرأ الكلمة السابقة؟

دوّن:
- هل القراءة تبدو كسطور منطقية؟
- هل يقرأ وصف العنصر فقط أم نصًا فعليًا من الصفحة؟
- هل يوجد تقطيع مزعج أو تكرار؟

## مرحلة 2: التنقل السريع بالحروف

اختبر على صفحة فيها عناصر متنوعة:
- `H` للعناوين
- `K` للروابط
- `E` لحقول الإدخال
- `B` للأزرار
- `X` لخانات الاختيار
- `D` للمعالم
- `T` للجداول
- `L` للقوائم
- `A` للحوارات أو التنبيهات
- `F` لعناصر النماذج

دوّن:
- هل ينتقل إلى العنصر الصحيح؟
- هل الاسم المعلن واضح؟
- هل `Shift+الحرف` يعود للخلف كما ينبغي؟

## مرحلة 3: وضع التصفح ووضع التركيز

اختبر:
1. انتقل إلى حقل إدخال باستخدام الأسهم أو `E`.
2. تأكد أن التطبيق يدخل الحقل تلقائيًا مع نغمة.
3. اخرج من الحقل مباشرة بالأسهم بدون كتابة.
4. تأكد أنه يعود إلى التصفح.
5. ادخل مرة أخرى واكتب نصًا.
6. تأكد أن الخروج التلقائي لا يحدث فورًا بعد بدء الكتابة.
7. استخدم `Insert+Space` يدويًا بين `browse mode` و`focus mode`.
8. استخدم `Escape` للرجوع إلى `browse mode`.

دوّن:
- هل النغمة واضحة؟
- هل الدخول التلقائي للحقل يحدث في الوقت الصحيح؟
- هل العودة للتصفح تعمل إذا لم تتم الكتابة؟
- هل يوجد تعارض بين الأسهم والكتابة؟

## مرحلة 4: النماذج

اختبر صفحة فيها:
- حقول نص
- خانات اختيار
- أزرار اختيار
- مربعات خيارات
- أزرار إرسال

تحقق من إعلان الحالات:
- `مطلوب`
- `غير صالح`
- `حالي`
- `تمت زيارته`
- `قيد التحديث`

دوّن:
- هل يعلن الحقل المطلوب بوضوح؟
- هل الأخطاء أو الحالات غير الصالحة تُقرأ؟
- هل عناصر النماذج المركبة مفهومة؟

## مرحلة 5: الجداول

اختبر صفحة فيها جدول حقيقي.

جرّب:
- `T` للوصول إلى الجدول
- `Insert+T` لقراءة سياق الجدول
- `Ctrl+Left / Ctrl+Right` للتنقل الأفقي
- `Ctrl+Up / Ctrl+Down` للتنقل العمودي

دوّن:
- هل يعلن اسم الجدول؟
- هل يعلن الصف والعمود الحاليين؟
- هل يعلن رؤوس الصفوف أو الأعمدة؟
- هل التنقل يفشل في بعض الجداول؟

## مرحلة 6: القوائم والحوارات

اختبر مواقع تحتوي على:
- قوائم تنقل
- قوائم جانبية
- نوافذ منبثقة
- رسائل تنبيه

جرّب:
- `L`
- `A`
- الأسهم بعد فتح الحوار

دوّن:
- هل يعلن الحوار أو القائمة بوضوح؟
- هل ينتقل إلى العنصر الصحيح؟

## مرحلة 7: live regions

اختبر صفحات تحتوي على:
- نتائج بحث تتحدث مباشرة
- رسائل نجاح أو خطأ
- تنبيهات فورية
- إشعارات تحميل أو تحديث

دوّن:
- هل تم نطق التحديث؟
- هل كان مناسبًا أم مزعجًا؟
- هل توجد رسائل مكررة؟
- هل الضجيج قليل بما يكفي؟

## مواقع مقترحة للاختبار

- صفحة ويكيبيديا أو مقال طويل
- نموذج تسجيل دخول
- نموذج بحث حي
- صفحة فيها جدول بيانات أو جدول أسعار
- متجر إلكتروني فيه رسائل `added to cart`
- واجهة ويب غنية مثل GitHub أو Gmail أو Outlook Web
