# خطة اختبار عملية

هذا المستند يساعد على اختبار Lumina يدويًا بعد كل build جديد على Windows.

## قبل البدء

1. شغّل Lumina.
2. افتح نافذة `Inspector` إذا كنت تريد مقارنة النطق بالأحداث المسجلة.
3. جرّب الأوامر الأساسية أولًا:
   - `Insert+F`
   - `Insert+Tab`
   - `Insert+Home`
   - `Insert+End`
   - `Insert+Q` داخل المحررات

## اختبار Windows Settings

1. افتح تطبيق `Settings`.
2. تنقل بين التبويبات أو الأقسام المختلفة.
3. تأكد أن النطق يميز بين:
   - التبويب
   - المجموعة
   - خانة الاختيار
   - زر الاختيار
   - مربع الخيارات
   - المنزلق
   - النص
4. جرّب:
   - `Insert+G`
   - `Insert+N`
   - `Insert+P`
   - `Insert+C`
   - `Insert+A`
   - `Insert+O`
   - `Insert+U`
   - `Insert+D`
   - `Insert+Z`
   - `Insert+X`
   - `Insert+J`
5. دوّن:
   - هل تم الانتقال إلى عنصر صحيح؟
   - هل تم تجاهل العناصر الزخرفية؟
   - هل يوجد تكرار مزعج أو أسماء غير مفهومة؟

## اختبار File Explorer

1. افتح `File Explorer`.
2. جرّب التنقل بين:
   - الملفات
   - المجلدات
   - شجرة التنقل
   - شريط العنوان
   - مربع البحث
3. افتح قائمة مثل `File` أو القائمة السياقية إن أمكن.
4. جرّب:
   - `Insert+F`
   - `Insert+Tab`
   - `Insert+M`
   - `Insert+N`
   - `Insert+P`
5. دوّن:
   - هل يميز بين ملف ومجلد؟
   - هل شجرة التنقل واضحة؟
   - هل أوامر القوائم تعطي مسارًا صحيحًا؟

## اختبار Notepad

1. افتح `Notepad`.
2. اكتب عدة أسطر عربية وإنجليزية.
3. حرّك المؤشر داخل النص.
4. جرّب:
   - `Insert+Up`
   - `Insert+Down`
   - `Insert+Left`
   - `Insert+Right`
   - `Insert+PageUp`
   - `Insert+PageDown`
   - `Insert+Q`
5. حدّد جزءًا من النص وأعد تجربة `Insert+Q`.
6. افتح مربع البحث أو الاستبدال إن أمكن.
7. دوّن:
   - هل رقم السطر والعمود منطقيان؟
   - هل التحديد يُقرأ بشكل صحيح؟
   - هل يميز بين المحرر وحقل البحث؟

## اختبار Notepad++

1. افتح `Notepad++`.
2. افتح ملفًا فعليًا أو أكثر.
3. تنقل بين التبويبات.
4. جرّب أوامر المراجعة النصية و`Insert+Q`.
5. افتح شريط البحث أو الاستبدال.
6. دوّن:
   - هل يعلن تبويب الملف؟
   - هل يميز بين المحرر وحقل البحث؟
   - هل شريط الحالة مفيد؟

## اختبار VS Code

1. افتح `VS Code`.
2. جرّب التنقل بين:
   - المحرر
   - المستكشف
   - البحث
   - الامتدادات
   - الطرفية
   - شريط الحالة
3. افتح مشروعًا فيه ملفات متعددة.
4. جرّب:
   - `Insert+F`
   - `Insert+Tab`
   - `Insert+Q`
   - `Insert+M` عند وجود قوائم
5. دوّن:
   - هل يميز بين `Explorer` و`Search` و`Extensions` و`Terminal`؟
   - هل التبويبات واضحة؟
   - هل أوصاف الألواح مناسبة أم عامة أكثر من اللازم؟

## اختبار المتصفح

1. افتح `Chrome` أو `Edge` أو `Firefox`.
2. افتح صفحة فيها:
   - عناوين
   - روابط
   - حقول إدخال
   - خانات اختيار
3. جرّب:
   - `Insert+T`
   - `Insert+W`
   - `Insert+S`
   - `H / Shift+H`
   - `K / Shift+K`
   - `E / Shift+E`
4. دوّن:
   - هل التنقل بين العناصر الدلالية صحيح؟
   - هل أسماء العناصر منطقية؟

## ملاحظات يجب تسجيلها

- اسم التطبيق
- العنصر الذي كان عليه التركيز
- الأمر المستخدم
- النتيجة المتوقعة
- النتيجة الفعلية
- هل المشكلة في:
  - النطق
  - تحديد الدور
  - التنقل
  - اختصار الأمر
  - تكرار/ضجيج
